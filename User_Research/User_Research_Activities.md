# User Research – Actividades y Justificación

## Sistema de Coordinación General de RRHH (CGRH – UADY)

### Contexto del proyecto

El sistema de Coordinación General de Recursos Humanos (CGRH) está diseñado para ser utilizado por todos los empleados de la Universidad Autónoma de Yucatán (UADY). Sin embargo, para efectos de este proyecto académico, el proceso de User Research se enfocará exclusivamente en empleados jubilados mayores de 60 años, quienes presentan características específicas como bajo o básico nivel de alfabetización digital, posible disminución visual y dificultades para recordar contraseñas.

Este enfoque permitirá analizar con mayor profundidad las necesidades de un grupo vulnerable y garantizar que el sistema cumpla con criterios de accesibilidad, simplicidad y facilidad de uso para funciones básicas como inicio de sesión y consulta de información financiera y administrativa (CFDI, Fondo de Ahorro para el Retiro y Caja de Ahorro).

---

## Actividades de User Research Seleccionadas

### 1. Interviews (Entrevistas)

**Justificación:**  
Las entrevistas se utilizarán para comprender las necesidades, limitaciones y expectativas de los empleados jubilados respecto al uso del sistema CGRH. Este método permitirá obtener información cualitativa detallada mediante un diálogo guiado, utilizando lenguaje claro y adaptado al perfil del usuario.

**Se espera obtener:**

- Principales dificultades al usar sistemas digitales
- Experiencia previa con plataformas institucionales
- Necesidades relacionadas con acceso y recuperación de contraseñas
- Expectativas generales sobre el sistema CGRH

---

### 2. Contextual Inquiry

**Justificación:**  
La observación contextual se aplicará para analizar cómo los usuarios interactúan con el sistema en su entorno real, ya sea desde una computadora personal o un dispositivo móvil Android básico. Este método permitirá identificar problemas que podrían no ser mencionados directamente por los usuarios durante entrevistas.

**Se espera obtener:**

- Información sobre el contexto real de uso
- Barreras físicas o cognitivas durante la interacción
- Dificultades relacionadas con visibilidad y navegación
- Dependencia de ayuda externa para completar tareas

---

### 3. Surveys (Encuestas)

**Justificación:**  
Las encuestas se emplearán para recopilar información general de un grupo más amplio de empleados jubilados, permitiendo identificar patrones comunes de uso y dificultades recurrentes. Las preguntas se diseñarán con lenguaje sencillo y estructura clara para facilitar su comprensión.

**Se espera obtener:**

- Dispositivos más utilizados por los usuarios
- Frecuencia de uso de las funciones del sistema
- Principales problemas reportados
- Nivel general de comodidad con el sistema

---

### 4. Usability Testing (Pruebas de Usabilidad)

**Justificación:**  
Las pruebas de usabilidad se llevarán a cabo con el objetivo de evaluar cómo los usuarios realizan tareas clave del sistema CGRH, tales como el registro, inicio de sesión, recuperación de contraseña y consulta de información financiera. Este método permitirá validar los requerimientos funcionales definidos para el sistema.

**Se espera obtener:**

- Errores más frecuentes durante tareas críticas
- Tiempo aproximado para completar cada tarea
- Problemas de comprensión de mensajes y botones
- Evidencia de problemas de accesibilidad y usabilidad

---

## Resultados esperados del proceso de User Research

- Identificación anticipada de problemas de usabilidad
- Comprensión clara de las necesidades del usuario objetivo
- Información suficiente para la creación de personas y escenarios
- Base sólida para el refinamiento de requerimientos funcionales
- Evidencia documentada para justificar decisiones de diseño
